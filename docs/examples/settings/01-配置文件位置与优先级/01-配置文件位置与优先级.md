# settings.json: 配置文件位置与优先级

> 指南 7.1「配置文件位置和优先级」— 4 个配置位置的查找和合并规则

## 目录结构

```
# 系统级（管理员配置）
/etc/claude-code/settings.json        ← 最高优先级，管理员设置

# 用户全局
~/.claude/settings.json               ← 用户个人全局配置

# 项目共享（提交 VCS）
<project>/.claude/settings.json       ← 项目团队共享配置

# 项目私有（不提交 VCS）
<project>/.claude/settings.local.json ← 项目个人私有配置
```

## 说明

```
4 个配置文件位置（优先级从低到高）:

  1. /etc/claude-code/settings.json    — 系统管理员配置
     适用: 企业环境统一管控
     由 IT 管理员设置，普通用户不能修改

  2. ~/.claude/settings.json           — 用户全局配置
     适用: 个人通用偏好
     所有项目共享，如全局 hooks、默认权限

  3. <project>/.claude/settings.json   — 项目共享配置
     适用: 团队统一规范
     提交到 Git，所有团队成员共享

  4. <project>/.claude/settings.local.json — 项目私有配置
     适用: 个人开发偏好
     不提交 Git（加入 .gitignore）

合并规则:
  → 所有层级的配置合并（不是替换）
  → 同名字段: 高优先级覆盖低优先级
  → 数组字段: 通常合并而非覆盖
  → 执行顺序: 系统 → 全局 → 项目 → 私有
```

## 使用示例

### 示例 1：四层配置叠加效果

```
# 1. /etc/claude-code/settings.json（系统管理员）
{
  "permissions": {
    "deny": ["Bash(rm -rf*)"],
    "defaultMode": "default"
  }
}

# 2. ~/.claude/settings.json（用户全局）
{
  "permissions": {
    "allow": ["Bash(git log*)", "Bash(git diff*)"],
    "deny": ["Bash(curl*|*)"]
  },
  "hooks": {
    "PreToolUse": [{"matcher": "Edit", "hooks": [{"type": "command", "command": "echo 'global hook'"}]}]
  }
}

# 3. .claude/settings.json（项目共享）
{
  "permissions": {
    "allow": ["Bash(npm test*)", "Bash(npm run build*)"]
  },
  "hooks": {
    "PostToolUse": [{"matcher": "Edit", "hooks": [{"type": "command", "command": "bash .claude/hooks/lint.sh"}]}]
  }
}

# 4. .claude/settings.local.json（项目私有）
{
  "theme": "dark",
  "verbose": true
}

→ 最终合并结果:
  permissions:
    allow: ["Bash(git log*)", "Bash(git diff*)", "Bash(npm test*)", "Bash(npm run build*)"]
    deny: ["Bash(rm -rf*)", "Bash(curl*|*)"]
    defaultMode: "default"
  hooks:
    PreToolUse: [全局 hook]
    PostToolUse: [项目 lint hook]
  theme: "dark"
  verbose: true
```

### 示例 2：项目级覆盖全局的 defaultMode

```
# ~/.claude/settings.json（全局）
{
  "permissions": {
    "defaultMode": "default"
  }
}
→ 所有项目默认需要手动确认操作

# .claude/settings.json（项目 A — 信任项目）
{
  "permissions": {
    "defaultMode": "acceptEdits"
  }
}
→ 项目 A 自动接受编辑操作

# .claude/settings.json（项目 B — 严格审核）
{
  "permissions": {
    "defaultMode": "plan"
  }
}
→ 项目 B 所有操作需要先制定计划

效果:
  → 全局设 default，项目可以覆盖
  → 不同项目不同安全策略
  → 个人项目用 acceptEdits，工作项目用 default
```

### 示例 3：项目私有配置不提交 VCS

```
# .gitignore 中添加:
.claude/settings.local.json
.claude/hooks/my-*.sh

# .claude/settings.local.json（个人配置）
{
  "permissions": {
    "allow": [
      "Bash(open *)",
      "Bash(code *)",
      "Bash(say *)"          // macOS 语音提醒
    ]
  },
  "hooks": {
    "Stop": [{
      "matcher": "",
      "hooks": [{
        "type": "command",
        "command": "osascript -e 'display notification \"Claude 完成了\" with title \"Claude Code\"'"
      }]
    }]
  }
}

→ git status 不显示 settings.local.json
→ 不影响其他团队成员
→ 每个人可以有独立的私有配置

效果:
  → 私有配置适合个人偏好
  → 不提交 VCS，不影响团队
  → .gitignore 确保不会被意外提交
```

### 示例 4：系统管理员配置统一管控

```
# /etc/claude-code/settings.json（IT 管理员设置）
{
  "permissions": {
    "deny": [
      "Bash(rm -rf *)",
      "Bash(curl *| *)",
      "Bash(wget * -O *)",
      "Bash(docker rm *)",
      "Bash(kubectl delete *)"
    ],
    "defaultMode": "default",
    "additionalDirectories": []
  },
  "mcpServers": {
    "company-registry": {
      "command": "npx",
      "args": ["-y", "@company/mcp-registry"],
      "env": {
        "REGISTRY_URL": "https://internal.company.com/mcp"
      }
    }
  }
}

→ 所有用户:
  → 不能执行危险命令（rm -rf, docker rm）
  → 默认需要手动确认（defaultMode: default）
  → 自动配置公司 MCP 服务器
  → 用户不能覆盖 deny 列表

效果:
  → 企业安全策略统一管控
  → 用户可以在允许范围内自定义
  → 系统级配置确保基础安全
```

### 示例 5：新项目初始化配置

```
→ 新项目启动:

# 步骤 1: 创建项目配置
mkdir -p .claude
cat > .claude/settings.json << 'EOF'
{
  "permissions": {
    "allow": [
      "Bash(git *)",
      "Bash(npm test*)",
      "Bash(npm run lint*)",
      "Bash(npm run build*)",
      "Read",
      "Glob",
      "Grep"
    ],
    "deny": [
      "Bash(npm publish*)",
      "Bash(docker push*)"
    ],
    "defaultMode": "default"
  },
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Edit|Write",
        "hooks": [
          {
            "type": "command",
            "command": "bash .claude/hooks/auto-lint.sh",
            "async": true
          }
        ]
      }
    ]
  }
}
EOF

# 步骤 2: 创建私有配置占位
cat > .claude/settings.local.json << 'EOF'
{
  "theme": "dark"
}
EOF

# 步骤 3: 添加到 .gitignore
echo ".claude/settings.local.json" >> .gitignore

# 步骤 4: 提交共享配置
git add .claude/settings.json .gitignore
git commit -m "chore: add Claude Code project settings"

效果:
  → 新项目有了标准的配置模板
  → 团队成员克隆后自动获得权限和 hooks
  → 私有配置在 .gitignore 中
```
