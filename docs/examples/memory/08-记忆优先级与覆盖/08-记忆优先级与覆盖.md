# Memory: 记忆优先级与覆盖

> 指南 8.1 + 8.2 — 多层记忆的优先级和冲突解决

## 说明

```
优先级（从低到高）:

  1. Auto Memory          → ~/.claude/memory/MEMORY.md
  2. 全局 CLAUDE.md        → ~/.claude/CLAUDE.md
  3. Agent Memory (user)   → ~/.claude/agent-memory/<type>/
  4. 项目 CLAUDE.md        → <project>/CLAUDE.md
  5. CLAUDE.local.md       → <project>/CLAUDE.local.md
  6. Agent Memory (project)→ .claude/agent-memory/<type>/
  7. 子目录 CLAUDE.md      → <subdir>/CLAUDE.md
  8. Agent Memory (local)  → .claude/agent-memory-local/<type>/

冲突解决: 高优先级覆盖低优先级
  项目 > 全局，local > project > user
```

## 使用示例

### 示例 1：Auto Memory vs 项目 CLAUDE.md

```
Auto Memory: "我喜欢用 arrow function"
项目 CLAUDE.md: "本项目使用 class 组件"

→ 项目 CLAUDE.md 优先 → 用 class 组件（本项目中）
→ Auto Memory 在其他项目仍然生效 → 用 arrow function
```

### 示例 2：Agent Memory user vs project scope

```
user scope: "审查时关注性能问题"
project scope: "本项目重点审查安全（金融项目）"

→ project scope 优先 → 重点审查安全
```

### 示例 3：子目录 CLAUDE.md 覆盖根目录

```
根 CLAUDE.md: "项目使用 REST API"
src/api/CLAUDE.md: "API 目录使用 tRPC"

→ 编辑 src/api/ 下文件 → 用 tRPC
→ 编辑其他文件 → 用 REST
```

### 示例 4：CLAUDE.local.md 覆盖 CLAUDE.md

```
CLAUDE.md: "测试覆盖率 > 80%"
CLAUDE.local.md: "本地暂时 > 60%（重构中）"

→ local 覆盖 → 60%（不影响团队）
```

### 示例 5：完整优先级链

```
编辑 src/api/auth/login.ts:

加载 9 层记忆:
  Auto Memory → TypeScript strict
  全局 CLAUDE.md → 函数式风格
  Agent Memory (user) → 关注安全
  项目 CLAUDE.md → NestJS + tRPC
  CLAUDE.local.md → 本地 DB 配置
  Agent Memory (project) → JWT + refresh token
  子目录 CLAUDE.md → rate limiting
  Agent Memory (local) → 正在重构 login
  rules/auth.md → 安全审查清单

→ 从通用到具体，层层精细
```
