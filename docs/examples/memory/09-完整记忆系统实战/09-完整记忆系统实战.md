# Memory: 完整记忆系统实战

> 指南 8.1 + 8.2 — 三种记忆协同工作的真实场景

## 使用示例

### 示例 1：新项目从零搭建记忆系统

```bash
# 1. 项目 CLAUDE.md（团队共享）
cat > CLAUDE.md << 'CLAUDE'
# 电商平台后端

## 技术栈
- Node.js 20 + TypeScript strict
- NestJS + Prisma + PostgreSQL

## 规范
- 函数式组件，不用 class
- API 输入用 Zod 验证
- 测试覆盖率 > 80%
CLAUDE

# 2. 私有配置（不提交）
cat > CLAUDE.local.md << 'LOCAL'
- 本地 DB: dev/dev@localhost:5432
- 用 pnpm
LOCAL

# 3. Auto Memory（个人偏好）
# /remember 我喜欢函数式编程风格

# 4. Agent + memory
cat > .claude/agents/migrator.md << 'AGENT'
---
description: Database migration specialist
memory: project
tools: [Read, Write, Edit, Bash]
---
AGENT

# 5. .gitignore
echo "CLAUDE.local.md" >> .gitignore
echo ".claude/agent-memory-local/" >> .gitignore
git add CLAUDE.md .claude/ .gitignore
```

### 示例 2：团队协作记忆共享

```
Alice（Team Lead）:
  编辑 CLAUDE.md → 添加团队规范
  git push → 团队获得

Bob（新成员）:
  git clone → CLAUDE.md 提供完整入职手册
  不需要口头说明规范

Agent Memory (project scope):
  通过 Git 共享 → 新成员自动获得 Agent 经验
```

### 示例 3：跨项目记忆复用

```
Alice 的 Auto Memory（跨项目）:
  - 偏好 TypeScript strict
  - 测试用 Vitest

Alice 在项目 A:
  Auto Memory + 项目 A 的 CLAUDE.md + Agent Memory

Alice 切换到项目 B:
  Auto Memory + 项目 B 的 CLAUDE.md + Agent Memory
  → 个人偏好跟随，项目规范切换
```

### 示例 4：记忆系统解决重复说明

```
没有记忆:
  每次都要重复 "用 TypeScript"、"用 pnpm"、"覆盖率 > 80%"

有记忆:
  CLAUDE.md: 项目规范（一次写好）
  Auto Memory: 个人偏好（/remember 一次）
  Agent Memory: Agent 经验（自动积累）
  → 零重复说明，直接开始工作
```

### 示例 5：记忆随使用自然积累

```
1 周后:
  Auto Memory: 5 条偏好
  Agent Memory: 2 个 Agent 有基础经验

1 个月后:
  Auto Memory: 15 条偏好
  Agent Memory: 5 个 Agent 有丰富经验
  项目 Memory: 完整的团队规范文档

→ 越用越智能
→ 知识积累不随会话结束丢失
→ Claude 成为真正的"老搭档"
```
