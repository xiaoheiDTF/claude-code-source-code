# Memory: 三种记忆类型总览

> 指南 8.1「三种记忆」— Auto Memory、Agent Memory、项目 Memory

## 目录结构

```
# Auto Memory（自动记忆）
~/.claude/
└── memory/
    └── MEMORY.md                  ← 当前用户，自动积累

# Agent Memory（Agent 记忆）
~/.claude/
└── agent-memory/
    └── code-reviewer/
        └── MEMORY.md              ← user scope，跨项目共享
<project>/.claude/
└── agent-memory/
    └── migrator/
        └── MEMORY.md              ← project scope，项目级
<project>/.claude/
└── agent-memory-local/
    └── personal-dev/
        └── MEMORY.md              ← local scope，本地

# 项目 Memory（项目记忆）
<project>/
├── CLAUDE.md                      ← 项目全局指令（团队共享）
├── CLAUDE.local.md                ← 项目私有指令（不提交）
└── .claude/
    └── rules/
        └── coding-style.md        ← 项目规则
```

## 说明

```
三种记忆类型对比:

  Auto Memory:
    路径: ~/.claude/memory/MEMORY.md
    作用: 自动积累用户偏好和习惯
    范围: 当前用户，所有项目
    写入: Claude 自动写入（/remember 命令）
    内容: "用户偏好 TypeScript strict mode"

  Agent Memory:
    路径: ~/.claude/agent-memory/<agentType>/MEMORY.md
    作用: 特定 Agent 类型的持久化记忆
    范围: 跨项目（user）/ 项目级（project）/ 本地（local）
    写入: Agent 运行时使用 Write 工具
    内容: "code-reviewer Agent 记住项目使用 ESLint 9"

  项目 Memory:
    路径: CLAUDE.md / .claude/rules/*.md
    作用: 项目级团队共享指令和规则
    范围: 当前项目
    写入: 手动编辑或 Claude 编辑
    内容: "本项目使用 pnpm，不使用 npm"
```

## 使用示例

### 示例 1：三种记忆同时存在于一个项目

```
~/.claude/memory/MEMORY.md（Auto Memory）:
  - 用户喜欢使用中文交流
  - 偏好函数式编程风格
  - 常用技术栈: React + Next.js + Prisma

~/.claude/agent-memory/code-reviewer/MEMORY.md（Agent Memory - user）:
  - 用户项目普遍使用 Vitest 测试框架
  - 安全审查重点: SQL 注入和 XSS

.claude/agent-memory/migrator/MEMORY.md（Agent Memory - project）:
  - 当前项目 schema 版本: 3.2.1
  - 最近迁移: add-user-email

CLAUDE.md（项目 Memory）:
  - 本项目使用 pnpm workspace monorepo
  - 代码风格: ESLint + Prettier
  - 提交规范: Conventional Commits

→ Claude 启动时加载所有记忆:
  Auto Memory → 知道用中文、函数式风格
  Agent Memory → code-reviewer 知道项目测试框架
  项目 Memory → 知道项目用 pnpm 和 monorepo
```

### 示例 2：不同记忆解决不同问题

```
问题: "帮我写一个新的 API 端点"

→ Auto Memory 帮助:
  "用户偏好 TypeScript strict mode" → 自动使用 strict 类型

→ Agent Memory 帮助:
  "api-gen Agent 记得上次生成的模式" → 复用之前的代码模式

→ 项目 Memory 帮助:
  "项目使用 tRPC 而不是 REST" → 生成 tRPC router 而不是 Express 路由

效果:
  → 三种记忆从不同维度影响 Claude 的行为
  → Auto Memory: 个人偏好
  → Agent Memory: Agent 经验积累
  → 项目 Memory: 团队规范
```

### 示例 3：Auto Memory 与项目 Memory 的区别

```
Auto Memory (memory/MEMORY.md):
  - 个人偏好: "我喜欢用 arrow function"
  - 通用习惯: "写代码时先写测试"
  - 跨项目通用: 适用于所有项目

项目 Memory (CLAUDE.md):
  - 项目规范: "本项目使用 class 组件，不用 hooks"
  - 团队约定: "API 路由必须加认证中间件"
  - 项目特有: 只适用于当前项目

→ 冲突时:
  Auto Memory: "我喜欢 arrow function"
  项目 Memory: "本项目使用 class 组件"
  → 项目 Memory 优先（项目规范 > 个人偏好）
```

### 示例 4：Agent Memory 与 Auto Memory 的区别

```
Auto Memory:
  → 所有 Claude 会话共享
  → 记录通用的用户偏好
  → 内容: "用户是 Python 后端开发者"

Agent Memory:
  → 只有特定类型的 Agent 共享
  → 记录 Agent 执行经验
  → 内容: "security-scanner Agent 记得项目 A 有 3 个已知漏洞"

→ Auto Memory 写入方式:
  用户: /remember 我喜欢用 TypeScript
  → 写入 ~/.claude/memory/MEMORY.md

→ Agent Memory 写入方式:
  Agent 运行时使用 Write 工具:
  Write: ~/.claude/agent-memory/code-reviewer/MEMORY.md
```

### 示例 5：记忆系统的生命周期

```
第一次使用 Claude Code:
  → 所有记忆为空
  → Claude 从零开始了解用户

使用 1 周后:
  Auto Memory 积累: 用户偏好、常用技术栈
  → Claude 逐渐适应用户风格

使用 1 个月后:
  Auto Memory: 丰富的个人偏好
  Agent Memory: 多个 Agent 的经验积累
  项目 Memory: 团队规范文档化
  → Claude 对项目和用户有深度理解

切换到新项目:
  Auto Memory: 继续生效（个人偏好跨项目）
  Agent Memory (user scope): 继续生效（跨项目）
  Agent Memory (project scope): 新项目没有（需要重新积累）
  项目 Memory: 新项目的 CLAUDE.md（新规范）
```
